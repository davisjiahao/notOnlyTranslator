## 用户故事拆分

### US-S2-1: 闪卡式单词复习 ⭐ P0

**作为**英语学习者，
**我希望**能够定期复习我标记过的生词，
**从而**通过闪卡方式强化记忆，提高词汇掌握度。

**验收标准**:
- [ ] 自动计算下次复习时间（艾宾浩斯算法）
- [ ] 闪卡界面美观，支持翻转动画
- [ ] 支持键盘快捷键（空格翻转，数字键评分）
- [ ] 复习记录持久化到 Chrome Storage
- [ ] 复习完成后的统计反馈

**技术要点**:
- 艾宾浩斯遗忘曲线算法实现
- Popup 页面集成或独立页面
- 与 MasteryManager 深度集成
- 本地通知 API 集成

**子任务**:
1. 设计闪卡数据结构和复习调度算法
2. 实现闪卡 UI 组件（正反面、翻转动画）
3. 集成艾宾浩斯算法计算复习时间
4. 添加键盘快捷键支持
5. 实现复习完成统计

## 技术方案概要

### 闪卡复习系统架构

```
Popup/Options Page
├── FlashcardReview Component
│   ├── CardDisplay (正面/背面)
│   ├── RatingButtons (评分 1-5)
│   └── ProgressIndicator
├── SpacedRepetition Algorithm
│   ├── calculateNextReview(sm2 or ebhinghaus)
│   └── updateWordMastery(word, rating)
└── ReviewScheduler
    ├── getDueWords()
    ├── scheduleNotification()
    └── syncWithStorage()
```

### 统计仪表盘架构

```
Options Page - Statistics Tab
├── SummaryCards
│   ├── TotalWordsCard
│   ├── MasteredWordsCard
│   └── LearningWordsCard
├── TrendCharts
│   ├── VocabularyGrowthChart (折线图)
│   ├── CEFRProgressChart (面积图)
│   └── LearningHeatmap (日历热力图)
└── TimeRangeSelector (7d/30d/90d/all)
```

## 风险与考虑

1. **艾宾浩斯算法**: 需要研究并实现合理的复习调度算法
2. **存储空间**: 统计历史数据可能占用较多 storage，需要考虑数据清理策略
3. **性能**: 大数据量的图表渲染需要优化
4. **向后兼容**: 新功能需要兼容旧版本存储的数据格式
