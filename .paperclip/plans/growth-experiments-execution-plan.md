## EXP-001: 欢迎弹窗流程优化

### 实验假设
优化后的欢迎弹窗能提升首次翻译成功率 25%。

### A/B 测试设计

| 组别 | 体验 | 目标 |
|------|------|------|
| A (对照) | 现有首次使用体验 | Baseline |
| B (实验) | 交互式教程 + 进度引导 | 提升转化率 |

### 关键改动点

1. **首次安装弹窗**
   - 欢迎语 + 核心价值主张
   - 简短教程 (3步骤)
   - "开始体验" CTA

2. **首次翻译引导**
   - 高亮功能按钮
   - 悬浮提示说明
   - 实时反馈

3. **完成奖励**
   - 首次翻译完成庆祝动画
   - "设置 API Key" 引导

### 成功指标

| 指标 | 目标 | 当前基线 |
|------|------|----------|
| 首次翻译完成率 | +25% | 待测量 |
| API Key 配置率 | +15% | 待测量 |
| 7日留存率 | +10% | 待测量 |

## EXP-002: 成就系统与社交分享

### 实验假设
成就系统能提升用户活跃度和推荐转化率 15%。

### 核心功能

| 成就 | 解锁条件 | 分享奖励 |
|------|----------|----------|
| 词汇小白 | 标记第一个生词 | 徽章 |
| 每日学习 | 连续3天使用 | 徽章+50积分 |
| 词汇达人 | 掌握100词 | 徽章+100积分 |
| 坚持不懈 | 连续30天 | 周边抽奖资格 |

### 技术需求

- 成就数据模型
- 成就解锁检测
- 分享卡片生成
- 邀请链接追踪

## EXP-003: API试用配额机制

### 实验假设
免费试用额度能降低门槛，提升7日付费转化率。

### 配额规则

| 用户类型 | 免费额度 | 说明 |
|----------|----------|------|
| 新用户 | 100次 | 一次性发放 |
| 邀请奖励 | 50次 | 被邀请人首次翻译后 |

### 防刷机制

- 设备限制: 每设备最多5次重置
- IP限制: 每日最多3次新用户
- 行为验证: 需完成至少一次翻译
