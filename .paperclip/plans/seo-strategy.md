# NotOnlyTranslator SEO 策略方案

## 方案时间
2026-03-15

## 目标
制定 NotOnlyTranslator 的搜索引擎优化策略，提升有机流量和品牌知名度。

---

## 一、项目现状分析

### 1.1 产品形态
- Chrome 浏览器扩展 (Manifest V3)
- 无独立官网/落地页
- 主要分发渠道：Chrome Web Store

### 1.2 SEO 挑战
1. Chrome 扩展难以获得传统搜索引擎排名
2. 依赖 Chrome Web Store 内部搜索和外部引流
3. 需要通过内容营销建立品牌认知

### 1.3 SEO 机会
1. 拦截目标用户搜索意图（"英语翻译插件推荐"）
2. 建立外部内容资产（博客、问答平台）
3. 技术文档和开发者社区覆盖

## 三、内容 SEO 策略

### 3.1 内容矩阵

| 内容类型 | 平台 | 目的 | 关键词 |

#### 主题 3：开发者工具
- "程序员如何高效阅读英文文档"
- "Stack Overflow 英文问答阅读技巧"
- "技术文档翻译工具推荐"

### 3.3 内容技术优化

**文章结构**:
```
H1: 核心主题
  H2: 子主题 1
    H3: 细节展开
  H2: 子主题 2
    H3: 细节展开
```

**SEO 元素**:
- Meta 标题：关键词 + 品牌
- Meta 描述：150 字内，包含行动号召
- 图片 Alt：描述性关键词
- 内链：相关文章链接
- 外链：权威来源引用

## 四、技术 SEO 策略

### 4.1 官网技术 SEO（未来）

如果未来需要建设官网：

| 检查项 | 要求 |
|--------|------|
| HTTPS | 强制启用 |
| 页面速度 | < 3s (LCP) |
| 移动友好 | 通过 Mobile-Friendly Test |
| 结构化数据 | Product, FAQ, Article |
| XML Sitemap | 提交 Search Console |
| robots.txt | 正确配置 |
| Canonical | 避免重复内容 |

### 4.2 技术 SEO 清单

#### 网站架构
- 清晰的导航结构
- 面包屑导航
- 逻辑性 URL 结构
- 404 页面自定义

#### 性能优化
- 图片压缩 (WebP)
- 懒加载
- 代码分割
- CDN 加速

#### 移动端
- 响应式设计
- 触摸友好
- 字体可读性

## 五、链接建设策略

### 5.1 外链建设

#### 高优先级
1. **技术博客**
   - 掘金、SegmentFault、CSDN

2. **问答平台**
   - 知乎回答相关问题
   - 锚文本：品牌名或功能描述

3. **导航目录**
   - Chrome 插件导航站
   - 产品目录提交

#### 中优先级
4. **教育类网站**
   - 考研社区、留学论坛
   - 软性植入

5. **开发者社区**
   - GitHub 项目页
   - Dev.to、Hashnode

### 5.2 内链策略

如果有多篇内容：
- 相关文章推荐
- 主题集群构建
- 锚文本多样性

### 5.3 社交信号

- 微博：话题参与
