## 一、目标用户画像分析

### 1.1 核心用户群体

| 用户类型 | 画像特征 | 痛点 | 增长策略重点 |
|---------|---------|------|-------------|
| **备考学生** | 四六级/考研/托福/雅思考生 | 英文阅读速度慢，反复查词打断思路 | 考试驱动，强调效率提升 |
| **职场白领** | 外企/跨境电商/翻译需求 | 专业术语查不到，邮件处理效率低 | 场景化，节省时间 |
| **终身学习者** | 程序员/研究者/阅读爱好者 | 缺乏系统性词汇积累方法 | 社区口碑，持续学习 |

### 1.2 用户获取优先级

```
第一优先级：备考学生（需求最刚性，传播意愿最强）
    ↓
第二优先级：职场白领（付费潜力高）
    ↓
第三优先级：终身学习者（口碑传播主力）
```

### 1.3 用户生命周期价值

```
新用户 → 激活用户 → 活跃用户 → 留存用户 → 推荐用户
   ↓         ↓          ↓          ↓         ↓
 安装      首翻      标记词     复习      邀请
```

## 三、推荐计划架构

### 3.1 推荐机制设计

**双向奖励体系**:

| 邀请人奖励 | 被邀请人奖励 | 条件 |
|-----------|-------------|------|
| 高级 API quota (100次) | 首周无限翻译 | 双方均需完成首次翻译 |
| 专属成就徽章 | 专属成就徽章 | 被邀请人完成7天连续学习 |
| 词汇本 pk 次数 | 挑战奖励 | 邀请3人后解锁 |

### 3.2 推荐流程

```
用户 A
  ↓ [邀请链接]
分享 → 社交媒体/好友
  ↓
用户 B (点击链接)
  ↓
安装 → 注册 → 首翻
  ↓
双方获得奖励
```

### 3.3 推荐追踪机制

- **唯一邀请码**: 每个用户生成唯一邀请码
- **链接追踪**: `?ref=USER_ID` 参数追踪来源
- **奖励发放**: 被邀请人完成关键行为后发放

### 3.4 防刷机制

| 机制 | 说明 |
|-----|------|
| 设备限制 | 每设备最多5次邀请奖励 |
| 行为验证 | 被邀请人需完成首次翻译 |
| 异常检测 | 同一IP/设备多次安装触发审查 |

## 五、转化率优化方案

### 5.1 用户引导 (Onboarding) 优化

**当前流程问题**:
1. 首次安装后缺乏明确指引
2. API Key 配置步骤繁琐
3. 英语水平设置缺乏紧迫感

**优化后流程**:
```
安装 → 欢迎弹窗(3秒) → 水平选择(10秒) → API配置(30秒) → 引导示例(60秒) → 完成
```

| 步骤 | 优化方案 | 预期效果 |
|------|---------|---------|
| 欢迎弹窗 | 3秒自动展示 + "立即体验"按钮 | 提升初始激活率 |
| 水平选择 | 预设考试选项 + "智能测试"按钮 | 降低决策门槛 |
| API 配置 | 提供免费试用 quota | 降低首次使用障碍 |
| 示例网站 | 内置 3 个英文网站快捷入口 | 快速体验价值 |

### 5.2 关键转化漏斗

```
展示 → 点击 → 安装 → 激活 → 留存 → 推荐
  ↓       ↓       ↓      ↓      ↓     ↓
商店页   下载    首次    7日    30日   邀请
```

**转化率目标**:

| 阶段 | 指标 | 目标值 |
|-----|------|--------|
| 展示→点击 | CTR | 15%+ |
| 点击→安装 | CVR | 8%+ |
| 安装→激活 | 激活率 | 50%+ |
| 激活→留存 | 7日留存 | 35%+ |
| 留存→推荐 | 分享率 | 10%+ |

### 5.3 A/B 测试计划

| 实验 | 假设 | 变量 | 指标 |
|-----|------|------|------|
| 欢迎弹窗 | 简化引导提升激活率 | 5步 vs 3步 | 7日激活率 |
| 成就分享 | 社交分享带来新用户 | 分享时机 | 邀请转化率 |
| API试用 | 降低付费障碍 | 有/无试用 | 首周付费率 |
| 截图顺序 | 价值展示提升安装 | 流程 vs 价值 | 安装转化率 |

| 事件 | 触发时机 | 用途 |
