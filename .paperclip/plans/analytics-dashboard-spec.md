## 二、核心指标定义

### 2.1 北极星指标

#### 日活跃用户 (DAU)

**定义**: 在24小时内至少完成一次翻译请求的唯一用户数

**计算公式**:
```
DAU = COUNT(DISTINCT user_id)
WHERE event = 'translation_request'
AND timestamp >= TODAY() - 1 DAY
AND timestamp < TODAY()
```

**分解公式**:
```
DAU = 新增用户 × 首日留存率 + 回流用户 + 持续活跃用户
```

**目标值**:
| 时间 | 目标DAU | 说明 |
|------|---------|------|
| 发布日 | 50 | 种子用户 |
| 1个月 | 500 | 初步验证 |
| 3个月 | 2,000 | 产品市场契合 |
| 6个月 | 5,000 | 规模增长 |

---

### 2.2 AARRR 指标体系

#### 获客 (Acquisition)

| 指标 | 定义 | 计算公式 | 目标值 |
|------|------|----------|--------|
| 新增安装 | 每日新安装用户数 | `COUNT(DISTINCT user_id WHERE install_date = TODAY)` | 1,000/周 |
| 商店曝光 | Chrome Web Store 展示次数 | Chrome开发者后台数据 | 20,000/周 |
| 商店CTR | 点击率 | `点击数 / 展示数` | 15%+ |
| 商店CVR | 安装转化率 | `安装数 / 访问数` | 8%+ |
| CAC | 获客成本 | `营销支出 / 新增用户数` | <$0.5 |

#### 激活 (Activation)

| 指标 | 定义 | 计算公式 | 目标值 |
|------|------|----------|--------|
| 激活率 | 完成首次翻译的用户比例 | `首次翻译用户数 / 安装用户数` | 50%+ |
| 7日激活率 | 7天内完成首次翻译的比例 | `7日内首次翻译数 / 安装数` | 40%+ |
| 设置完成率 | 完成初始设置的比例 | `完成设置数 / 开始设置数` | 70%+ |
| 设置时长 | 从安装到完成设置的时间 | AVG(设置完成时间 - 安装时间) | <3分钟 |
| 首次翻译时长 | 从安装到首次翻译的时间 | AVG(首次翻译时间 - 安装时间) | <24小时 |

#### 留存 (Retention)

| 指标 | 定义 | 计算公式 | 目标值 |
|------|------|----------|--------|
| 次日留存 | 第2天仍活跃的用户比例 | `第2天活跃数 / 首日安装数` | 45%+ |
| 7日留存 | 第7天仍活跃的用户比例 | `第7天活跃数 / 首日安装数` | 35%+ |
| 30日留存 | 第30天仍活跃的用户比例 | `第30天活跃数 / 首日安装数` | 20%+ |
| 周活跃用户 | 7天内至少活跃1次的用户数 | `COUNT(DISTINCT user_id WHERE days_active_last_7 > 0)` | 3,500+ |
| 月活跃用户 | 30天内至少活跃1次的用户数 | `COUNT(DISTINCT user_id WHERE days_active_last_30 > 0)` | 10,000+ |

**留存曲线定义**:
```
Day N 留存率 = Day 0 安装的用户中，Day N 仍活跃的比例

活跃用户定义：当天至少完成一个核心行为
- 核心行为：翻译请求 / 标记词汇 / 查看生词本
```

#### 收入 (Revenue) - 未来

| 指标 | 定义 | 计算公式 | 目标值 |
|------|------|----------|--------|
| ARPU | 每用户平均收入 | `总收入 / 总用户数` | - |
| 付费转化率 | 付费用户比例 | `付费用户数 / 总用户数` | 5%+ |
| MRR | 月度经常性收入 | `SUM(月订阅费用)` | - |
| LTV | 用户生命周期价值 | `AVG(用户总付费额)` | >3×CAC |

#### 推荐 (Referral)

| 指标 | 定义 | 计算公式 | 目标值 |
|------|------|----------|--------|
| 分享率 | 主动分享的用户比例 | `分享用户数 / 活跃用户数` | 10%+ |
| 邀请转化率 | 邀请链接带来的安装比例 | `邀请安装数 / 邀请链接点击数` | 15%+ |
| 病毒系数 (K) | 每个用户带来的新用户数 | `邀请安装数 / 邀请人总数` | >0.3 |
| NPS | 净推荐值 | `推荐者% - 贬损者%` | >30 |

## 八、附录

### 8.1 SQL 查询示例

**DAU计算**:
```sql
SELECT
  DATE(timestamp) as date,
  COUNT(DISTINCT user_id) as dau
FROM events
WHERE event_name = 'translation_request'
  AND DATE(timestamp) >= DATE_SUB(CURRENT_DATE(), INTERVAL 30 DAY)
GROUP BY 1
ORDER BY 1
```

**留存计算**:
```sql
WITH user_cohorts AS (
  SELECT
    user_id,
    DATE(install_date) as cohort_date
  FROM users
),
daily_activity AS (
  SELECT
    user_id,
    DATE(timestamp) as active_date
  FROM events
  WHERE event_name = 'translation_request'
  GROUP BY 1, 2
)
SELECT
  c.cohort_date,
  DATE_DIFF(d.active_date, c.cohort_date, DAY) as day_n,
  COUNT(DISTINCT d.user_id) as active_users,
  COUNT(DISTINCT c.user_id) as cohort_size,
  ROUND(
    COUNT(DISTINCT d.user_id) * 100.0 / COUNT(DISTINCT c.user_id),
    2
  ) as retention_rate
FROM user_cohorts c
LEFT JOIN daily_activity d
  ON c.user_id = d.user_id
  AND d.active_date >= c.cohort_date
GROUP BY 1, 2
ORDER BY 1, 2
```
