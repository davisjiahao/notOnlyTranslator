## 二、不同社群类型运营 SOP

### 2.1 新人欢迎群（7天新手引导）

#### 入群流程
```
Day 1 - 欢迎 + 产品介绍
- 发送欢迎语和群规则
- 产品介绍视频/图文
- 新人任务：完成一次翻译体验

Day 2 - 功能教学
- 核心功能演示
- 常见问题解答
- 新人任务：设置个人水平

Day 3 - 学习资料
- 学习资料包发放
- 学习方法分享
- 新人任务：制定学习计划

Day 4 - 社群互动
- 破冰游戏
- 学习伙伴配对
- 新人任务：分享学习心得

Day 5 - 问题答疑
- 1对1答疑时间
- 个性化建议
- 新人任务：提交反馈

Day 6 - 进阶引导
- VIP权益介绍
- 学习社群推荐
- 新人任务：选择下一步社群

Day 7 - 毕业仪式
- 学习证书颁发
- 成果展示
- 迁移到正式社群
```

### 2.2 学习打卡群（每日打卡）

#### 打卡流程
```
1. 每日打卡模板：
   【Day X 打卡】
   📚 今日学习内容：xxxx
   ⏱️ 学习时长：xx分钟
   💡 收获/感悟：xxxx
   🤔 疑问：xxxx（选填）

2. 打卡监督机制：
   - 连续3天未打卡 @提醒
   - 连续7天未打卡 私信关怀
   - 连续15天未打卡 询问是否退出

3. 打卡激励：
   - 7天连续打卡：电子徽章
   - 30天连续打卡：实体礼品
   - 100天连续打卡：VIP会员
```

### 2.3 VIP付费群（深度服务）

#### 服务标准
```
1. 响应时效：
   - 工作日 2小时内回复
   - 周末/节假日 4小时内回复
   - 紧急问题 30分钟内响应

2. 专属服务：
   - 每月1次1对1学习诊断
   - 每周专属学习资料推送
   - 新功能优先体验权
   - 产品问题优先处理

3. 社群活动：
   - 每周专家分享会
   - 每月线下聚会（一线城市）
   - 季度学习成果展示
```

## 三、用户分层运营 SOP

### 3.1 标签体系

| 标签类型 | 标签内容 | 用途 |
|----------|----------|------|
| **基础信息** | 性别/年龄/地域/职业 | 用户画像、内容推送 |
| **来源渠道** | 小红书/抖音/产品内/老带新 | 渠道效果分析 |
| **用户分层** | 新用户/活跃用户/沉默用户/付费用户 | 差异化运营 |
| **兴趣标签** | 考研/四六级/职场英语/雅思 | 精准内容推荐 |
| **行为标签** | 高频打卡/偶尔互动/从未发言 | 活跃度评估 |
| **价值分层** | KOC/VIP/普通用户 | 资源倾斜分配 |

### 3.2 分层运营策略

#### 新用户（关注7天内）
```
目标：建立信任、引导首次使用

触达频率：
- Day 1: 欢迎语 + 产品简介
- Day 2: 功能教程推送
- Day 3: 学习资料包
- Day 5: 使用回访
- Day 7: 社群邀请

内容方向：
- 产品价值传递
- 使用教程
- 新手福利

关键指标：
- 首次使用率 > 60%
- 7日留存率 > 40%
```

#### 活跃用户（近7天有互动）
```
目标：提升粘性、促进转化

触达频率：
- 每日：社群内容/打卡提醒
- 每周：精选内容汇总
- 每月：学习报告/活动邀请

内容方向：
- 深度干货
- 社群活动
- VIP权益介绍

关键指标：
- 月活跃度 > 50%
- 内容互动率 > 10%
```

#### 沉默用户（30天未互动）
```
目标：唤醒激活、重新参与

触达频率：
- Week 1: 关怀消息 + 福利提醒
- Week 2: 新功能/新内容推送
- Week 3: 专属优惠
- Week 4: 问卷调研/退出确认

内容方向：
- 唤醒福利
- 新功能介绍
- 用户关怀

关键指标：
- 唤醒率 > 15%
- 取关率 < 5%
```

## 五、数据监测 SOP

### 5.1 每日数据检查清单

```
□ 新增好友数：目标 vs 实际
□ 新增社群数
□ 社群活跃度（发言人数/总用户数）
□ 私信回复率
□ 用户反馈数量
□ 异常事件记录
```

### 5.2 每周数据分析

| 指标 | 计算方式 | 健康基准 |
|------|----------|----------|
| 新增好友周增长率 | (本周新增-上周新增)/上周新增 | > 10% |
| 社群活跃率 | 活跃用户数/总用户数 | > 15% |
| 7日留存率 | 7天后仍活跃的新用户/总新用户 | > 40% |
| 私信转化率 | 成交用户数/咨询用户数 | > 5% |
| 用户满意度 | 好评数/总评价数 | > 90% |

### 5.3 数据异常处理流程

```
发现数据异常
    ↓
分析原因（对比历史数据/检查运营动作）
    ↓
制定应对策略
    ↓
执行调整
    ↓
监测效果（连续观察3-7天）
    ↓
记录备案
```

## 六、常见问题处理 SOP

### 6.1 用户投诉处理

| 投诉类型 | 处理时效 | 处理流程 |
|----------|----------|----------|
| 产品使用问题 | 2小时内 | 1. 道歉并表达理解<br>2. 询问具体问题<br>3. 提供解决方案<br>4. 跟进确认解决<br>5. 记录反馈给产品 |
| 服务态度问题 | 1小时内 | 1. 真诚道歉<br>2. 了解具体情况<br>3. 补偿方案（优惠券/礼品）<br>4. 内部通报改进<br>5. 回访确认满意 |
| 退费/退款 | 4小时内 | 1. 了解退费原因<br>2. 尽力挽留/解决问题<br>3. 如坚持退费，说明流程<br>4. 快速办理<br>5. 记录原因分析 |

### 6.2 危机事件处理

#### 负面舆情
```
发现负面舆情
    ↓
评估影响范围（24小时内监测）
    ↓
制定回应策略
    ↓
统一对外口径
    ↓
官方渠道回应
    ↓
持续监测 + 主动沟通核心用户
    ↓
复盘总结
```

#### 账号异常
```
发现账号异常（被封/限流等）
    ↓
立即启用备用账号
    ↓
通知用户迁移
    ↓
申诉主账号
    ↓
分析原因 + 调整策略
    ↓
更新应急预案
```
