# 如何安装和配置 NotOnlyTranslator？

> 从 Chrome Web Store 安装到个性化配置，这篇指南带你完整走一遍 NotOnlyTranslator 的初始设置流程。

---

## 安装步骤

### 1. 从 Chrome Web Store 安装

1. 打开 Chrome 浏览器，访问 Chrome Web Store
2. 在搜索框输入 "NotOnlyTranslator"
3. 找到插件后点击「添加至 Chrome」
4. 在弹出的确认框中点击「添加扩展程序」

**快捷链接**: [Chrome Web Store - NotOnlyTranslator](https://chrome.google.com/webstore/detail/notonlytranslator/xxx?utm_source=zhihu&utm_medium=answer&utm_campaign=install_guide)

### 2. 固定扩展图标

安装完成后，建议将扩展图标固定在工具栏：

1. 点击 Chrome 右上角「扩展程序」图标（拼图形状）
2. 找到 NotOnlyTranslator，点击右侧的「固定」图钉
3. 现在图标会一直显示在地址栏右侧

---

## 初始配置：5分钟完成设置

### 第一步：词汇水平评估

首次点击扩展图标，会进入引导流程：

1. **开始评估** - 点击「评估我的词汇量」
2. **完成测试** - 系统会展示 50 个词汇，你只需标记「认识」或「不认识」
3. **查看结果** - 测试完成后，会显示你的预估词汇量（如 CET-6 水平、雅思水平等）

**提示**：评估结果不需要很精确，后续使用中系统会根据你的反馈自动调整。

### 第二步：选择翻译引擎

NotOnlyTranslator 支持多种 AI 翻译引擎：

| 引擎 | 特点 | 适用场景 |
|------|------|----------|
| **OpenAI GPT-4o-mini** | 速度快，成本低 | 日常使用首选 |
| **Anthropic Claude** | 翻译质量高 | 长句、复杂语境 |
| **自定义 API** | 灵活配置 | 有自建 API 的用户 |

**配置方法**：
1. 点击扩展图标 → 选择「设置」
2. 在「翻译引擎」选项卡中选择提供商
3. 输入你的 API Key（可在对应官网获取）
4. 点击「保存」

**获取 API Key**：
- OpenAI: [platform.openai.com](https://platform.openai.com)
- Anthropic: [console.anthropic.com](https://console.anthropic.com)

### 第三步：调整翻译触发方式

根据个人习惯选择触发方式：

| 触发方式 | 说明 | 推荐人群 |
|----------|------|----------|
| **悬停显示** | 鼠标悬停在生词上显示翻译 | 喜欢快速浏览的用户 |
| **点击显示** | 点击生词才显示翻译 | 希望减少干扰的用户 |

设置路径：扩展图标 → 设置 → 触发方式

### 第四步：主题设置

支持浅色/深色模式：
- 可跟随系统自动切换
- 也可手动指定主题

---

## 开始使用

配置完成后，打开任意英文网页：

1. **自动识别** - 页面加载后，插件会自动识别超出你词汇水平的单词
2. **高亮显示** - 生词会以虚线下划线标注
3. **查看翻译** - 悬停或点击生词，弹出翻译卡片
4. **标记掌握程度** - 查看翻译后，可以标记为「认识」或「不认识」

---

## 进阶配置

### 自定义词汇水平

如果你对自己的词汇量有清晰认知，可以手动调整：

1. 设置 → 词汇水平 → 自定义
2. 选择词表（四六级/雅思/托福/考研）
3. 设定具体水平

### 隐私模式

如果担心数据安全，可以开启隐私模式：

- 开启后，翻译请求直接发送到 AI 服务商，不经过中间服务器
- 生词本等数据仅存储在本地浏览器

设置路径：设置 → 隐私 → 本地模式

### 快捷键设置

为常用操作设置快捷键（Chrome 扩展管理 → 键盘快捷键）：

- 开关插件: `Alt+T`
- 打开生词本: `Alt+V`
- 刷新页面翻译: `Alt+R`

## 常见问题

**Q: 安装后没有反应？**
A: 请刷新页面后再试，或检查是否开启了隐私模式限制。

**Q: 词汇评估结果不准？**
A: 没关系，评估只是起点。使用过程中标记「认识/不认识」，系统会自动校准。

**Q: 某些网站不生效？**
A: 部分网站（如 Chrome 商店页面、Google Docs）限制内容脚本注入，这是浏览器安全策略，无法绕过。

**Q: API Key 安全吗？**
A: API Key 存储在你的浏览器本地，不会上传到任何服务器。
