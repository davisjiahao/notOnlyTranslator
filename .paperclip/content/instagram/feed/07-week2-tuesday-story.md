```
作为一名程序员
我每天需要阅读大量的英文文档

但我发现：
❌ 全页翻译 = 读的是中文，学不到英语
❌ 划词翻译 = 不断打断阅读节奏
❌ 查词典 = 在多个标签页间来回切换

有没有一种工具...
能只翻译我不认识的词？
```

### Slide 3 - 洞察

```
💡 关键洞察

真正的问题不是
"翻译不够快"

而是
"翻译太多了"

当你知道某个词的意思
你就不需要翻译
当你不知道
你才需要帮助

所以，我们需要的是
"个性化翻译"
```

### Slide 4 - 解决方案

```
✨ NotOnlyTranslator 诞生了
```

### Caption

```
💭 为什么我要做一个"不那么好用"的翻译工具？

说实话，市面上已经有很多优秀的翻译工具了。
Google翻译、DeepL、沉浸式翻译...
它们都很棒。

但问题是：它们都在解决"怎么翻译"，
却忽略了"怎么学"。

作为一个需要每天阅读英文内容的程序员，
我太了解这个痛点了。

📖 当我读一篇文章时：
- 90%的词我都认识 → 不需要翻译
- 10%的词不认识 → 需要翻译

但现有的工具要么全翻译（学不到东西），
要么不翻译（看不懂内容），
要么让我不断划词查询（打断阅读）。

所以我决定自己做一个工具：

✨ NotOnlyTranslator
只翻译你不认识的词

它不是来替代现有翻译工具的
而是来填补"个性化学习"这个空白的
```
