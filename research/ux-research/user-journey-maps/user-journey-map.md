# 用户旅程地图 (User Journey Map)

## 用户画像：进阶英语学习者 "小李"

---

## 旅程阶段 1：认知阶段 (Awareness)

### 用户场景
小李在阅读英文技术博客时，遇到了一些生词，需要频繁查词典，感到阅读体验很不流畅。

### 用户行为
- 在浏览器中阅读英文技术文章
- 遇到生词时切换到词典网站查询
- 尝试记忆生词但缺乏系统性

### 用户想法
- "查词太麻烦了，每次都要切换窗口"
- "这些词我下次遇到可能还是不认识"
- "有没有更好的学习工具？"

### 情感曲线
😕 困惑 → 😤 沮丧 → 🤔 好奇

### 痛点
- 频繁切换窗口打断阅读流畅性
- 查词效率低，影响阅读体验
- 缺乏系统性的词汇学习方法

### 机会点
- 浏览器插件形式，无需切换窗口
- 智能识别难度，只翻译需要帮助的词
- 提供单词标记和复习功能

---

## 旅程阶段 2：考虑阶段 (Consideration)

### 用户场景
小李在 Chrome 应用商店搜索"翻译插件"，发现了 NotOnlyTranslator，开始了解产品功能。

### 用户行为
- 浏览 Chrome 应用商店
- 查看 NotOnlyTranslator 介绍和截图
- 阅读用户评价
- 对比其他翻译插件（如沉浸式翻译）

### 用户想法
- "这个插件的理念很有意思，根据我的水平来翻译"
- "看起来界面很简洁，不会干扰阅读"
- "有词汇学习功能，这正是我需要的"

### 情感曲线
🧐 好奇 → 😊 感兴趣 → 🤔 犹豫

### 痛点
- 担心插件会拖慢浏览器速度
- 不确定翻译准确度如何
- 担心隐私问题（API key）

### 机会点
- 强调轻量级设计，不影响性能
- 展示用户好评和翻译示例
- 说明 API key 由用户自己保存，安全可靠

---

## 旅程阶段 3：安装与首次使用 (Onboarding)

### 用户场景
小李决定尝试安装 NotOnlyTranslator，开始设置和使用。

### 用户行为
1. 点击"添加至 Chrome"安装插件
2. 打开插件图标，进入设置页面
3. 配置 API key（或跳过稍后配置）
4. 选择英语水平（CEFR 等级）
5. 访问英文网页测试功能

### 用户想法
- "安装很简单，一步到位"
- "需要 API key...这个设置有点复杂"
- "英语水平选择很贴心，让我选择自己的水平"
- "哇，只翻译了我真正需要帮助的词！"

### 情感曲线
😊 满意 → 😕 困惑 → 🤩 惊喜

### 痛点
- API key 配置对新用户来说有门槛
- 不确定自己的 CEFR 等级
- 首次使用可能需要一点学习时间

### 机会点
- 提供更详细的 API key 获取指南
- 添加 CEFR 等级测试或说明
- 提供交互式教程，引导用户了解功能
- 添加示例页面，让用户立即体验效果

---

## 旅程阶段 4：日常使用 (Daily Use)

### 用户场景
小李已经将 NotOnlyTranslator 纳入日常英语学习流程，每天使用插件阅读英文内容。

### 用户行为
- 每天阅读英文技术博客时使用插件
- 遇到生词时查看翻译
- 将重要词汇标记为"已知"或"未知"
- 定期复习标记的单词（使用闪卡功能）
- 偶尔查看词汇量统计

### 用户想法
- "现在阅读英文文章流畅多了"
- "标记功能帮助我系统地积累词汇"
- "能看到自己的进步，很有成就感"
- "闪卡功能很实用，利用碎片时间复习"

### 情感曲线
😌 舒适 → 😊 满意 → 🎉 成就感

### 痛点
- 偶尔会遇到翻译不准确的情况
- 词汇量统计可能不够精确
- 希望能导出学习数据

### 机会点
- 持续优化翻译质量
- 提供更多统计维度和可视化
- 添加数据导出功能
- 开发更多学习模式（如拼写练习）

---

## 旅程阶段 5：高级使用与推荐 (Advocacy)

### 用户场景
小李已经成为 NotOnlyTranslator 的重度用户和推荐者，积极向他人推荐这款插件。

### 用户行为
- 在社交媒体上分享使用体验
- 向同事和朋友推荐插件
- 在 Chrome 应用商店留下好评
- 参与用户反馈，提出功能建议
- 可能参与 Beta 测试新功能

### 用户想法
- "这个插件真的改变了我的英语学习方式"
- "我要推荐给所有想提升英语的同事"
- "希望能看到更多新功能"
- "如果能支持更多语言就更好了"

### 情感曲线
🌟 忠诚 → 🚀 热情 → 🎊 倡导

### 痛点
- 希望有更多高级功能
- 可能希望获得早期访问新功能的机会
- 期待更个性化的功能

### 机会点
- 建立用户社区，促进用户交流
- 设立 Beta 测试项目
- 开发高级/专业版功能
- 创建推荐奖励计划
- 定期举办线上/线下用户活动

---

## 旅程地图总结

### 关键触点 (Key Touchpoints)

1. **Chrome 应用商店** - 发现和安装
2. **插件图标/弹出窗口** - 日常交互
3. **Options 设置页面** - 配置和个性化
4. **翻译工具提示 (Tooltip)** - 核心功能体验
5. **统计仪表盘** - 进度追踪
6. **闪卡复习界面** - 学习功能

### 情感低谷点

1. **API Key 配置** - 技术门槛
2. **首次使用学习** - 新用户适应期
3. **翻译不准确** - 功能期望落差

### 高光时刻

1. **首次体验智能翻译** - "只翻译我需要帮助的词"
2. **词汇量进步可视化** - 成就感
3. **日常阅读流畅度提升** - 核心价值体现

### 改进优先级

**高优先级**：
1. 优化 API Key 配置流程，提供更详细的引导
2. 添加交互式新手教程
3. 持续优化翻译质量

**中优先级**：
4. 提供更多个性化设置选项
5. 增强数据导出功能
6. 开发社区功能

**低优先级**：
7. 多语言支持扩展
8. 游戏化学习元素
9. 用户成就系统
