# 可用性测试方案

## 测试概览

| 项目 | 详情 |
|------|------|
| **测试目标** | 评估新用户首次使用 NotOnlyTranslator 的易用性 |
| **测试类型** | 任务型可用性测试 |
| **测试人数** | 5 人 |
| **测试时长** | 45-60 分钟/人 |
| **测试形式** | 远程视频会议 + 屏幕共享 |
| **测试工具** | Chrome 浏览器 + NotOnlyTranslator 扩展 |

---

## 测试目标

### 主要目标
1. **评估任务完成率**：用户能否成功完成核心任务
2. **测量任务完成时间**：完成各步骤所需时间
3. **识别可用性问题**：发现让用户困惑或卡住的地方
4. **收集定性反馈**：了解用户的主观感受和期望

### 测试范围

**包含的功能**:
- 扩展安装后的首次使用
- API Key 配置流程
- 首次翻译操作
- 单词标记功能
- 设置调整

**不包含的功能**:
- 高级设置
- 数据导出
- 统计分析

---

## 测试任务设计

### 任务 1：安装和初步了解（5-7 分钟）

**场景设定**:
> 您刚刚安装了 NotOnlyTranslator 扩展，准备开始第一次使用。

**任务指令**:
1. 请点击浏览器工具栏上的扩展图标
2. 看看弹出的界面，您看到了什么？
3. 根据这个界面，您觉得这个扩展是做什么的？

**观察要点**:
- [ ] 用户是否能找到扩展图标
- [ ] 用户是否能理解 Popup 界面内容
- [ ] 用户对产品的第一印象
- [ ] 用户是否能理解产品的用途

**成功标准**:
- 用户成功打开 Popup
- 用户能基本理解产品用途

---

### 任务 2：配置 API Key（15-20 分钟）

**场景设定**:
> 要使用翻译功能，您需要配置一个 API Key。我们会提供一个测试用的 API Key。

**任务指令**:
1. 请尝试配置 API Key
2. 使用我们提供的测试 Key：`test-api-key-for-ux-research`
3. 配置完成后，告诉我们结果

**观察要点**:
- [ ] 用户是否能找到配置入口
- [ ] 用户是否能理解 API Key 的作用
- [ ] 配置过程中的错误次数
- [ ] 用户在哪里遇到困难
- [ ] 用户是否阅读了配置说明

**成功标准**:
- 成功配置 API Key
- 显示"配置成功"提示

**提供帮助的时机**:
- 用户卡住超过 2 分钟
- 用户明确求助

---

### 任务 3：进行首次翻译（10-15 分钟）

**场景设定**:
> 现在我们来试用一下翻译功能。请打开这个英文网页：
> https://en.wikipedia.org/wiki/Main_Page

**任务指令**:
1. 在这个页面上使用扩展翻译一些英文内容
2. 尝试翻译一个段落的英文
3. 告诉我们您是如何操作的

**观察要点**:
- [ ] 用户是否能找到翻译功能入口
- [ ] 用户使用的翻译方式（悬浮窗/右键/快捷键）
- [ ] 翻译响应时间
- [ ] 用户对翻译结果的反应
- [ ] 用户是否能理解翻译结果

**成功标准**:
- 成功翻译英文内容
- 能看到翻译结果

---

### 任务 4：标记单词（8-10 分钟）

**场景设定**:
> 如果您遇到一个不认识的单词，可以标记它以便日后复习。

**任务指令**:
1. 请标记一个您不认识的单词
2. 查看已标记的单词列表
3. 尝试查看这个单词的详细信息

**观察要点**:
- [ ] 用户是否能找到标记功能
- [ ] 标记操作是否直观
- [ ] 用户是否能找到已标记的单词列表
- [ ] 用户对词汇学习功能的理解

**成功标准**:
- 成功标记单词
- 能找到标记单词列表

---

### 任务 5：探索设置（5-8 分钟）

**场景设定**:
> 如果您想调整一些设置，比如更改目标语言或调整翻译样式。

**任务指令**:
1. 请尝试找到设置页面
2. 尝试更改一个设置项（如目标语言）
3. 看看还有哪些可以设置的内容

**观察要点**:
- [ ] 用户是否能找到设置入口
- [ ] 设置页面是否易于理解
- [ ] 用户是否能成功更改设置
- [ ] 用户对设置选项的理解程度

**成功标准**:
- 找到设置页面
- 成功更改一个设置项

## 评分标准

### 任务成功率

| 等级 | 描述 | 评分 |
|------|------|------|
| 成功 | 用户独立完成任务，无帮助 | 3 分 |
| 轻微帮助 | 用户完成任务，需要少量提示 | 2 分 |
| 大量帮助 | 用户完成任务，需要大量帮助 | 1 分 |
| 失败 | 用户未能完成任务 | 0 分 |

### 问题严重程度

| 等级 | 描述 | 定义 |
|------|------|------|
| 🔴 严重 | 阻碍任务完成 | 用户无法完成任务，或需要大量帮助 |
| 🟠 中等 | 造成显著延迟或困惑 | 用户需要额外时间或提示才能继续 |
| 🟡 轻微 | 造成轻微不便 | 用户注意到问题但能轻松解决 |
| 🟢 建议 | 改进建议 | 不影响任务完成，但可提升体验 |

## 测试脚本

### 开场白（3-5 分钟）

```
您好，感谢您参与这次可用性测试！我是 NotOnlyTranslator 的用户研究员。

【说明目的】
今天我们要测试的是产品的易用性，不是测试您。如果您在使用过程中遇到困难，那是产品设计的问题，不是您的问题。

【测试流程】
测试大约需要 45-60 分钟。我会给您布置几个任务，您尝试完成就好。请在使用过程中"边做边说"，告诉我您在做什么、在想什么、有什么疑问。

【屏幕共享】
请开启屏幕共享，这样我可以看到您的操作过程。

【录音】
为了便于后续分析，我会录制这次测试。录音仅用于产品改进，不会外传。

【随时提问】
如果您有任何问题，随时可以问我。不过我会尽量在任务完成后再回答，除非您卡住了。

您准备好开始了吗？还有什么问题吗？
```

### 任务引导语

**每个任务开始前**:
```
下一个任务：【任务描述】

场景：【场景设定】

请尝试：【具体任务】
```

**任务进行中**:
- 保持安静，让用户独立操作
- 记录时间、错误次数、用户表情和言语
- 只在用户求助或卡住超过 2 分钟时提供帮助

**任务完成后**:
```
好的，这个任务完成了。

- 刚才您在完成这个任务时，感觉如何？
- 有哪里让您感到困惑吗？
- 您期望的操作方式和实际有什么不同？
```

### 结束语（5-8 分钟）

```
所有任务都完成了。非常感谢您的参与！

现在请您填写一下这个简短的问卷，大概需要 2-3 分钟。

【问卷填写后】

感谢您的反馈！这对我们改进产品非常有帮助。

【发送奖励】
我们会在 24 小时内将 30 元京东卡发送到您的邮箱。

【最后提问】
- 您还有什么想要补充的吗？
- 对于我们的研究，您有什么建议吗？

再次感谢！祝您使用愉快！
```

## 测试参与者招募

可用性测试参与者可以与用户访谈参与者部分重叠，但建议至少 3-5 名新参与者。

**招募标准**: 与用户访谈相同
**激励方式**: 30元京东卡 或 1个月高级会员
**时间安排**: 访谈完成后 2-3 天进行

## 数据分析方法

### 定量分析

1. **任务成功率计算**
   ```
   成功率 = (所有任务总分 / 满分) × 100%
   ```

2. **SUS 分数计算**
   - 奇数题：得分 - 1
   - 偶数题：5 - 得分
   - 总分 × 2.5 = SUS 分数

3. **任务时间分析**
   - 计算每个任务的平均完成时间
   - 识别异常值（时间过长/过短）

### 定性分析

1. **问题聚类**
   - 将发现的问题按类型分类
   - 识别高频出现的问题

2. **优先级排序**
   - 根据严重程度和出现频率排序
   - 🔴 严重问题优先解决

3. **改进建议**
   - 针对每个问题提出改进方案
   - 评估改进的可行性和影响
