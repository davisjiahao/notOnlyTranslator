## 竞品 1：沉浸式翻译 (Immersive Translate)

### 产品定位
- 国产双语网页翻译扩展领导者
- 主打 AI 驱动的沉浸式翻译体验
- 支持多翻译引擎（OpenAI、DeepL、Google、微软等）

### 核心功能分析

| 功能类别 | 具体功能 | 用户价值 |
|----------|----------|----------|
| 翻译模式 | 网页双语、段落翻译、PDF翻译 | 全场景覆盖 |
| 翻译引擎 | 10+ 引擎支持，用户可选 | 灵活性和控制权 |
| 学习功能 | 生词高亮、单词本 | 语言学习辅助 |
| 视频支持 | YouTube/Netflix双语字幕 | 娱乐学习场景 |

### 交互设计亮点

```
┌─────────────────────────────────────────────┐
│ 1. 悬浮球交互                                │
│    - 页面右下角常驻                          │
│    - 一键开关翻译                            │
│    - 快速切换引擎                            │
├─────────────────────────────────────────────┤
│ 2. 双语对照显示                              │
│    - 原文下方直接显示译文                     │
│    - 段落级对照，阅读流畅                     │
├─────────────────────────────────────────────┤
│ 3. 智能缓存                                  │
│    - 翻译结果自动缓存                        │
│    - 重复内容秒出结果                        │
└─────────────────────────────────────────────┘
```

### 性能优化策略

| 策略 | 实现方式 | 效果 |
|------|----------|------|
| 多引擎并行 | 同时请求多个引擎，取最快结果 | 降低延迟 |
| 智能缓存 | 相同内容不重复翻译 | 节省 API 调用 |
| 本地处理 | 简单规则本地处理 | 即时响应 |

### 用户痛点

1. **功能复杂度高** - 新手容易 overwhelmed
2. **设置分散** - 配置项分布在多处
3. **无互动教程** - 依赖文字说明

## 竞品 2：Relingo

### 产品定位
- 单词学习 + 翻译结合的工具
- 主打"阅读中自动学习单词"
- 基于用户英语水平智能高亮生词

### 核心功能分析

| 功能类别 | 具体功能 | 用户价值 |
|----------|----------|----------|
| 智能高亮 | 根据CEFR级别高亮生词 | 个性化学习 |
| 翻译引擎 | DeepL/Bing/OpenAI | 质量保证 |
| 视频学习 | YouTube/播客双语字幕 | 沉浸式学习 |
| 单词复习 | Anki算法间隔重复 | 长期记忆 |

### 核心差异化功能：智能生词发现

```javascript
// Relingo 的核心算法逻辑（推测）
const userLevel = 'B1'; // 用户当前水平
const wordLevel = getCEFRLevel(word); // 单词难度

if (wordLevel > userLevel) {
  highlight(word); // 高亮生词
  showTranslation(word); // 显示翻译
}
```

### Onboarding 流程（4步）

| 步骤 | 用户操作 | 产品响应 |
|------|----------|----------|
| 1 | Pin扩展图标 | 确认安装成功 |
| 2 | 设置语言水平 | 初始化生词阈值 |
| 3 | 打开任意英文网页 | 自动高亮生词 |
| 4 | 自定义主题等 | 完成个性化设置 |

### UX 亮点

1. **无干扰设计** - 生词高亮不影响阅读流畅度
2. **渐进式学习** - 单词在后续阅读中重复出现
3. **即时反馈** - 点击生词立即显示翻译
4. **进度追踪** - 可视化学习进度

## 翻译速度优化分析

### 当前实现状态

根据代码审查，NotOnlyTranslator 已实现以下优化：

```
src/background/
├── hybridTranslation.ts      # 混合翻译服务 ✅
├── textComplexityAnalyzer.ts # 文本复杂度分析 ✅
├── deeplTranslation.ts       # DeepL集成 ✅
├── llmEnhancedAnalysis.ts    # LLM增强分析 ✅
└── enhancedCache.ts          # 增强缓存 ✅
```

### 智能路由决策逻辑

```typescript
// 当前实现
complexity <= 35  → 传统翻译 (DeepL/Youdao)  // 快速
complexity >= 65  → LLM翻译                   // 高质量
35 < complexity < 65 → 并行翻译               // 平衡
```

### 性能对比分析

| 翻译方案 | 平均延迟 | 翻译质量 | 成本 | 适用场景 |
|----------|----------|----------|------|----------|
| 纯 LLM | 2-5秒 | ⭐⭐⭐⭐⭐ | 高 | 复杂文本、学习场景 |
| 纯传统 | 200-500ms | ⭐⭐⭐ | 低 | 简单文本、快速浏览 |
| 混合方案 | 500ms-2秒 | ⭐⭐⭐⭐ | 中 | 全场景 |

### 优化建议

#### 1. 用户感知优化

当前问题：用户不清楚翻译使用了哪个引擎

**建议**：添加翻译来源指示器

```
┌─────────────────────────────────────┐
│ "Hello world"                       │
│ ↓                                   │
│ "你好世界" [DeepL · 0.3s]          │  ← 显示引擎和耗时
└─────────────────────────────────────┘
```

#### 2. 预翻译策略

```
页面加载 → 扫描所有文本 → 后台预翻译简单词汇
                                   ↓
                          用户点击时即时显示
```

#### 3. 优先级队列

```
高优先级：可视区域内的文本 → 立即翻译
低优先级：滚动区域外的文本 → 延迟翻译
```

#### 2. 智能生词发现（对标 Relingo）

```typescript
interface VocabularyProfile {
  knownWords: Set<string>;      // 已知词汇
  learningWords: Set<string>;   // 学习中词汇
  userLevel: CEFRLevel;         // A1-C2

  // 智能判断是否高亮
  shouldHighlight(word: string): boolean {
    if (this.knownWords.has(word)) return false;
    const wordLevel = getCEFRLevel(word);
    return wordLevel > this.userLevel;
  }
}
```
