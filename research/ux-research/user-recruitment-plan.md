# 用户访谈招募方案

## 招募概览

| 项目 | 详情 |
|------|------|
| **研究目标** | 了解新用户首次使用 NotOnlyTranslator 的体验 |
| **招募人数** | 8-10 人 |
| **目标用户** | 近7天内安装扩展的新用户 |
| **访谈时长** | 30-45 分钟 |
| **激励方式** | 30元京东卡 / 1个月高级会员 |
| **访谈形式** | 腾讯会议 / Zoom / Google Meet |
| **记录方式** | 录音 + 屏幕共享（经用户同意） |

---

## 招募标准

### 必备条件（必须全部满足）

1. **时间要求**
   - 在过去 7 天内安装了 NotOnlyTranslator 扩展
   - 安装后至少打开过一次 Popup

2. **用户类型**
   - 实际有英文阅读需求（工作、学习或兴趣）
   - 能够清晰表达使用体验

3. **设备要求**
   - 使用 Chrome 或 Edge 浏览器
   - 能够进行视频通话和屏幕共享
   - 网络稳定

### 优先招募（满足越多越好）

1. **用户群体多样性**
   - [ ] 开发者/技术人员（2-3人）
   - [ ] 学生（2-3人）
   - [ ] 职场人士（2-3人）
   - [ ] 其他领域（1-2人）

2. **使用经验多样性**
   - [ ] 成功完成 API 配置并使用了翻译功能（3-4人）
   - [ ] 尝试配置但未成功（2-3人）
   - [ ] 尚未尝试配置（2-3人）

3. **英语水平多样性**
   - [ ] 初级（CET-4 或同等）
   - [ ] 中级（CET-6 或同等）
   - [ ] 高级（专业八级或留学经验）

### 排除条件

- 扩展开发团队成员或其亲友
- 其他翻译扩展的开发者
- 无法使用普通话或英语进行交流的用户
- 无法保证 30 分钟完整时间的用户

## 访谈安排流程

### Step 1: 筛选（收到报名后 24 小时内）

1. 查看问卷回复
2. 根据招募标准筛选
3. 优先选择多样性强的用户
4. 发送知情同意书

### Step 2: 确认（收到同意回复后 24 小时内）

1. 确认用户符合所有条件
2. 发送访谈时间选项（提供 3 个时间段）
3. 确认访谈方式（视频会议/语音）
4. 发送会议链接

### Step 3: 提醒（访谈前）

**前 1 天**:
- 发送提醒邮件/消息
- 确认会议链接
- 提醒准备事项（稳定的网络、安静的环境）

**前 1 小时**:
- 再次确认是否准时参加

### Step 4: 访谈（30-45 分钟）

1. 开场（5 分钟）：自我介绍、说明流程、确认录音许可
2. 主体（25-35 分钟）：按照访谈大纲进行
3. 结束（5 分钟）：总结、感谢、说明奖励发放

### Step 5: 奖励发放（访谈后 24 小时内）

1. 发送京东卡卡密 或
2. 激活高级会员资格
3. 发送感谢邮件

## 风险预案

### 招募人数不足

**应对措施**:
1. 扩大招募渠道（增加社交媒体投放）
2. 提高激励金额（50元京东卡）
3. 放宽时间限制（近14天内安装）
4. 延长招募周期

### 用户爽约

**应对措施**:
1. 招募时多招募 2-3 人作为备选
2. 提前 1 天和 1 小时双重确认
3. 建立候补名单

### 访谈质量问题

**应对措施**:
1. 访谈中发现用户不符合条件，提前结束并寻找替补
2. 录音质量差时，立即重录关键部分
3. 准备标准化追问问题库
