## 问题说明

经过技术评估，**无法直接提供"新用户邮箱列表"**，原因如下：

### 技术限制

1. **无用户邮箱收集机制**
   - NotOnlyTranslator 是浏览器扩展，无后端服务器
   - 所有用户数据存储在本地 Chrome Storage
   - 从未收集过用户邮箱地址

2. **隐私设计原则**
   - 扩展遵循最小数据收集原则
   - 用户数据完全本地化，不上传到任何服务器
   - 无法追踪或识别具体用户身份

---

## 替代方案

### 方案 1: 通过招募表单收集邮箱（推荐）

**修改 Google Forms，添加邮箱字段**：

```
报名表单字段:
1. 您的邮箱地址 (必填)
2. 您安装扩展的大致时间
3. 是否有英文阅读需求
4. 是否已配置 API Key
5. 其他联系方式（微信/电话）
```

**优点**:
- 用户主动提供，符合隐私法规
- 筛选意愿强的用户
- 可验证用户确实安装了扩展

**实施**: UX Researcher 修改表单即可

---

### 方案 2: 应用内数据导出功能

**在扩展中添加"参与用户研究"按钮**：

```typescript
// 在 Popup 或 Options 页面添加
<button onClick={exportResearchData}>
  参与用户研究
</button>

// 导出匿名数据 + 用户手动填写邮箱
{
  "usageData": { /* 匿名使用数据 */ },
  "email": "用户手动填写",
  "consent": true
}
```

**优点**:
- 可附带匿名使用数据（词汇量、使用频率等）
- 用户明确同意参与研究
- 数据质量高

**实施**: 需要 1-2 天开发时间

---

### 方案 3: 通过 Chrome Web Store 评价回复

**在 Chrome Web Store 回复最新评价，邀请用户参与**：

```
回复模板:
"感谢您的评价！我们正在进行用户研究，诚邀您参与 30 分钟访谈，
 提供 30元京东卡作为报酬。如有兴趣，请通过以下链接报名：
 [Google Forms 链接]"
```

**优点**:
- 零开发成本
- 用户是活跃用户

**缺点**:
- 无法筛选新用户
- 回复可能被忽略

---

## 建议

**推荐采用方案 1**（修改表单收集邮箱），原因：

1. **最快实施**: 30 分钟即可完成
2. **合规**: 用户主动提供，符合 GDPR/隐私法规
3. **有效**: Banner 引导用户到表单，表单收集邮箱
