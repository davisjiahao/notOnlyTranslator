## 技术决策确认

### 混合翻译架构
```
用户请求
    ↓
复杂度分析 (TextComplexityAnalyzer)
    ↓
智能路由
    ├── simple → DeepL (快速、低成本)
    ├── medium → Hybrid (DeepL + LLM分析)
    └── complex → LLM (高质量、完整功能)
    ↓
翻译结果
    ↓
缓存 (EnhancedCacheManager，差异化过期)
```

### 缓存策略
| 来源 | 过期时间 | 理由 |
|------|----------|------|
| DeepL | 30天 | 翻译结果稳定，重复利用价值高 |
| Hybrid | 14天 | 中等稳定性 |
| LLM | 7天 | 结果可能变化，保持新鲜度 |

### 回退策略
1. 主策略失败 → 记录错误
2. 尝试备用翻译引擎
3. 所有引擎失败 → 返回原文（保证可用性）

**主动行动**: 发现并修复 mastery.ts 贝叶斯更新逻辑 bug

**问题描述**:
- 4 个测试失败位于 `mastery.test.ts`
- 根本原因：认识高难度单词反而获得更低的掌握度

**修复内容**:

| 修改 | 文件 | 描述 |
|------|------|------|
| 难度因子逻辑 | `src/shared/utils/mastery.ts` | 引入难度因子 (0.5-1.0)，难度越高奖励越大 |

**原逻辑问题**:
```
expectedP = estimatedVocabSize / (estimatedVocabSize + wordVocabPosition)
// 低难度词: expectedP = 0.4 → targetP = 0.5
// 高难度词: expectedP = 0.2 → targetP = 0.3
// 结果：认识高难度词获得更低掌握度！
```

**修复后逻辑**:
```
difficultyFactor = 0.5 + (wordDifficulty / 10) * 0.5  // 0.5-1.0
targetP = priorMastery + difficultyFactor * 0.3
// 高难度词获得更高的掌握度奖励
```
